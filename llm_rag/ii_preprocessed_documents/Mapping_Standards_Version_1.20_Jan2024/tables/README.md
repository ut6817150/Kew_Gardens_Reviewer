# Mapping_Standards_Version_1.20_Jan2024/tables

## Purpose

This folder contains CSV exports of tables extracted from
`Mapping_Standards_Version_1.20_Jan2024.pdf`.

The CSV files are generated by `llm_rag/ii_preprocessed_documents/preprocess_pdfs.py`.
They are inspection artifacts and provenance companions for the row-level
records in the corresponding `retrieval_blocks.jsonl` file.

## How These Tables Are Used

For each extracted table, preprocessing writes:

- one parent `table` retrieval record containing the wider table context
- one `table_row` retrieval record per extracted row
- one CSV file in this folder for inspection

The vector-db build keeps table rows atomic so precise row-level mapping
questions can retrieve exact coding or attribute evidence.

## Notes

PDF table extraction is lossy. Some column names are generic, such as
`column_4`, because the original PDF table layout does not always provide clean
machine-readable headers. These CSVs should be treated as retrieval assets, not
as publication-ready spreadsheets.
