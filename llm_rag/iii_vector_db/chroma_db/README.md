# chroma_db

## Purpose

`llm_rag/iii_vector_db/chroma_db/` stores persistent Chroma database assets for
the reference retriever.

This folder is generated by:

```bash
python llm_rag/iii_vector_db/build_reference_db.py --reset
```

It is not Python cache. It is part of the retrieval asset set used by dense
reference retrieval.

## Current Layout

```text
llm_rag/iii_vector_db/chroma_db/
|- README.md
`- reference_docs/
```

`reference_docs/` is the persistent Chroma collection directory for the current
reference corpus.

## Relationship To Other Vector Assets

The vector-db stage has both dense and non-dense assets:

- `chroma_db/reference_docs/`
  Persistent dense index used by Chroma.
- `reference_corpus.jsonl`
  Sparse retrieval corpus used by the local BM25-style index.
- `parent_contexts.jsonl`
  Larger table contexts attached to row-level retrieval hits.
- `thresholds.json`
  Deterministic threshold facts used before or alongside retrieval.

Dense retrieval can be skipped if embeddings or Chroma are unavailable at query
time, but sparse retrieval and threshold lookup can still operate from the JSON
assets.

## Rebuilding

Rebuild this folder when:

- raw PDFs change
- preprocessing output changes
- chunking settings change
- embedding model settings change
- the dense index appears stale or corrupted

Run from the repo root:

```bash
python llm_rag/iii_vector_db/build_reference_db.py --reset
```

Do not delete this folder as routine cache cleanup unless you plan to rebuild
it before running dense retrieval.
